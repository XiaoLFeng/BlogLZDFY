基于PicGo搭建稳定的图床（支持GitHub）
发表于2020-08-07|更新于2020-08-12|实用技术教程
|阅读量:|评论数:

好像最近更新挺频繁的，可能时间多了作业少了（个鬼、死ね） Emmm，为什么我用Github做图床呢？为什么，我也不知道反正我就是这样玩。而且我以前也说了用Github做图床，因为可以使用jsdelivr作为CDN节点使用，老快了堪比开飞机。还是全球的图床（误）。

嘛，其实不只是只能用GitHub的，其实很多都可以用。例如免费的SM.MS图床，腾讯云COS，微博图床[不提供了]，GitHub图床，七牛图床，Imgurl图床，阿里云OSS，又拍云图床，以及增加插件会有很多的效果。当然，这里只展示Github搭建过程。

你可能需要的

PicGo项目地址：https://github.com/Molunerfinn/PicGo/releases 软件使用文档：https://picgo.github.io/PicGo-Doc/zh/guide/config.html 首先下载软件啦：下载地址见上“PicGo项目地址”，如果是Windows系统找到「.exe」下载即可。 或者这里哦 http://s.frp.xiaolfeng.xyz/#s/6F3lefCQ 然后自己安装总会吧，安装吧。这里用[kbd]Windows 10[/kbd]做演示 既然是GitHub图床，那么必须创建一个库（不会创的隔壁百度咯） 建议将 Initialize this repository with a README 点开，你也可以不点开，就是后期自己上传一个README.md如果嫌麻烦就直接点开吧~ 然后打开设置，我们需要设置一个Token（令牌） 然后我们给这个Token分配权限，这些勾代表对应有的权限，下面打勾就足以够了，如果全部打钩无妨，但是要[mark]保证安全性[/mark]（我不建议这么做，小心你号没了） 然后拉到最下面看到绿色的按钮点下去~ :mao7: 额，注意一下，这个代码你最好保存一下，他只会显示一次，以后忘了只能重置，所以保存一下奥~ 然后打开你的软件开始设置吧。[mark]打开软件→图床设置→GitHub图床[/mark]进入设置 H)1A7X{(0K.png) 首先仓库名可以在这里找，如下图所示即为仓库名 对于[kbd]设定分支名[/kbd]说明：这个嘛，一般就是master如果你没有做特殊设置 对于[kbd]设定Token[/kbd]说明：你刚刚创建的Token那一串代码 对于[kbd]设定自定义域名[/kbd]说明：

  • 首先jsdelivr的CDN节点IP为 https://cdn.jsdelivr.net/
  • 然后在后面添加 gh/
  • 然后在后面添加你的仓库名 XXXXX/XXXXX
  • （注意上一个后面不带“/”）然后添加 @master/ 就是你最后的CDN节点，属于这个仓库名的CDN节点
  • 也就是说连起来就是 https://cdn.jsdelivr.net/gh/XXXXX/XXXXX@master/
  • 例如我的 https://cdn.jsdelivr.net/gh/XiaoLFeng/ImageGo@master/
  • 然后填入自定义域名即可（注意不要把最后面的“/”带上，只是我们访问需要带上）
  • 举个例子，我填入就是 https://cdn.jsdelivr.net/gh/XiaoLFeng/ImageGo@master

然后点击保存，然后上传一个文件试试吧~
